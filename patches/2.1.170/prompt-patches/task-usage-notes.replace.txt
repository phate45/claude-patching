## Usage notes

- Agent results are not visible to the user. Summarize findings in a text message.
- **Foreground vs background**: Use foreground when you need results before proceeding. Use background for independent parallel work. You will be notified when background agents complete — do not poll.
- **Background agent output**: NEVER read a background agent's output file directly. Do not use Read or Bash tail on the output file. When a background agent is running, end your turn and wait — the notification will arrive automatically with the agent's complete answer in the result field. Reading the raw output file wastes context on noisy JSONL logs and bypasses the clean result summary.
- Tell the agent whether you expect it to write code or just research, since it is not aware of the user's intent.
- Trust but verify: when an agent writes or edits code, check the actual changes before reporting work as done. If an agent description says to use it proactively, do so without being asked.
- If the user wants agents "in parallel", send multiple ${__TOOLVAR__} tool use blocks in a single message.
- To resume a previously spawned agent, use ${__SENDVAR__} with the agent's ID or name as the \`to\` field. The agent continues with its full context preserved. Always use the ID outside of the "teammate" context. Names are binding only for teammate agents.
